## 🔄 Memory Sync (At Every Session Start)
> MEMORY.md is Claude's **temporary buffer (inbox)**. `_claude/` is the sole permanent store (Single Source of Truth).
> Content that Claude Code automatically writes to MEMORY.md during a session is classified and transferred to `_claude/` at the next session start.

### Principles
- **MEMORY.md** = Temporary buffer (working memo between sessions). Keep minimal
- **`_claude/`** = Permanent store. The source of truth for all structured knowledge
- No duplicates: Do not keep information in MEMORY.md if it already exists in `_claude/`

### Process (Once at every session start, before all other routines)
1. Read MEMORY.md
2. Classify each item:
   | Content Type | Target File |
   |-----------|-----------|
   | User preferences/dislikes | `_claude/3_knowledge/preferences.md` |
   | Tool/service changes | `_claude/2_context/tools-stack.md` |
   | Project status changes | `_claude/2_context/current-focus.md` |
   | Important decisions | `_claude/3_knowledge/decisions-log.md` |
   | Lessons/insights | `_claude/3_knowledge/lessons-learned.md` |
   | Profile changes | `_claude/1_identity/profile.md` |
   | Work style changes | `_claude/1_identity/work-style.md` |
   | Communication preference changes | `_claude/1_identity/communication-pref.md` |
   | Claude feedback | `_claude/4_interaction/feedback-log.md` |
   | Behavioral rules (for Claude) | Suggest reflecting in CLAUDE.md itself |
3. Items already reflected in `_claude/` → skip
4. New items → Reflect in target file (save after user confirmation)
5. Clean up MEMORY.md: Remove completed items, keep only the previous session's working state

### Triggers
- **Automatic at every session start** (runs before all other routines)
- User keywords: "memory cleanup", "memory sync", "sync"

## 📅 Daily Morning Routine — Daily Morning Review (Automatic)
> **Every day** at session start, automatically execute the logic below.
> Also runs when the user says "today's tasks", "TOP3", "morning review", "daily review", etc.

### Triggers
- **Automatic at the first session each day**
- User keywords: "today's tasks", "TOP3", "morning review", "daily", "morning routine", "check tasks"

### Execution Logic

#### Step 1: Check/Create Today's Daily Note
- Check if `00_데일리/{today's date}.md` exists
- If not, auto-create based on `30_참고자료/템플릿/Daily Note Template.md`

#### Step 2: Collect Today's Task Candidates (Scan 5 Sources)
1. **Yesterday's daily note** → Check `Tomorrow's Priority` section (carry-over tasks)
2. **`current-focus.md`** → Incomplete (`[ ]`) items that can be addressed today
3. **`01_인박스/`** → Items with `status: inbox` that are due soon or quick to complete
4. **`10_단발 프로젝트/`** → Next actions for projects approaching their deadline
5. **`30_참고자료/업무매뉴얼/월간루틴체크리스트.md`** → Today's applicable routines (if file exists)

#### Step 3: Suggest TOP3
- Sort collected candidates by priority and **suggest TOP3**:
  - 🔴 **Due soon** (due today or tomorrow) → Highest priority
  - 🟡 **Needs to be done this week** → Important
  - 🟢 **Nice to work on** → When time allows
- After user confirmation/edits, record in the daily note's `Today's Focus` section

#### Step 4: Today's Appointments/Meetings Reminder
- Remind about events listed in the daily note's `Appointments & Meetings`
- Check `current-focus.md`'s `Upcoming Schedule` for today's items

### Evening Review (Before Session Ends)
> Run when the session has been long, or when the user requests "wrap up for today", "evening review", "summarize"

1. Update the `Completed Today` section in today's daily note
2. Check incomplete items → Suggest whether to carry over to tomorrow
3. Write `Tomorrow's Priority` (seed for tomorrow's TOP3)
4. Suggest whether `current-focus.md` needs updating

## 🔁 Monday Routine Check (Automatic)
> **Every Monday** at session start, automatically execute the logic below.

1. Read `30_참고자료/업무매뉴얼/월간루틴체크리스트.md` (skip if file doesn't exist)
2. Based on today's date, extract tasks applicable to **this week (Mon-Fri)**:
   - "Weekly recurring" → Always included
   - "Monthly recurring" → Included if the date range overlaps
   - "Quarterly/Semi-annual/Annual" → Included if it's the applicable month
   - "Ad-hoc check" → Only included during the first week of the month (1st-5th)
3. Present extracted items sorted by **priority**:
   - 🔴 Due soon (due within this week)
   - 🟡 Needs to be progressed this week
   - 🟢 Items to review/prepare
4. Even on non-Mondays, run the same logic if the user says "this week's routines", "routine check", "check tasks", etc.

### PARA (Projects, Areas, Resources, Archive) Structure
- 10_단발 프로젝트/ = Projects (tasks with a start and end)
- 20_지속 관리 업무/ = Areas (domains requiring ongoing management)
- 30_참고자료/ = Resources (reference knowledge)
- 40_완료 프로젝트 및 업무/ = Archive (completed/inactive)

### Front Matter Format
Apply the following YAML front matter to all task notes:
- status: inbox / active / delegated / someday / done
- created: date
- gtd_type: pending / project / area / reference / someday / delegated
- due: deadline
- delegated_to: delegate target
- tags: []

→ Execute everything in order. Start from Step 1!
